Smoothed-particle hydrodynamics

Smoothed-particle hydrodynamics(SPH) is a method for simulating fluid flows. In SPH, a fluid is represented by a discrete set of particles. Each particle stores information such as mass, position, and velocity and together they represent scalar and vector fields of simulated field quantites of the fluid. At each timestep, each particle interacts with each other particle simulating the forces of nature and new positions/velocities are calculated. Utilizing the Navierâ€“Stokes equations to describe the motion of fluid flows, the forces from viscosity, pressure, and gravity are simulated on each particle pair.

In a thesis by Mattias Linde, he explores scaling the standard SPH algorithm to run on a parallel distributed system. As more particles are introduced into a simulation to increase the volume or resolution of the fluid, simulated interactions between millions of particles becomes computationally intensive. To divide the work among multiple machines, the problem is spatially decomposed into boxes that contain particles. Each box is assigned to a separate machine. Each machine simulates particle interaction in parallel, communicating information about neighboring particles with each other when necessary. In order to load balance the problem, box boundaries are redrawn at each timestep of the simulation to maintain an balanced number of particles per box.

The fluid simulation was run on a 192 node cluster with AMD 248 Opteron CPU's and 8GB of RAM per node, far from the computing capacities of machines on the Top 500 list. In a 200,000 particle system, the simulation took 9762.5 seconds to finish on a single machine and 578.64 seconds on 25 machines. Increasing computing 25x only increased performance 16.8x representing 67% efficiency being achieved on the 25 machine cluster.

The parallel algorithm scaled reasonably well on multiple machines. When analyzing how much time was spent at each stage of the algorithm, Linde found that 40% of the time was spent on communicating particle information between machines. Because, particles need to commmunicate information about neighboring particles before being able to simulate fluid flow with the Naiver-Stokes equations, the communication step acts as a natural barrier. Linde hypothesized that because this communication is lightweight, perhaps some machines were stuck in the communication stage for a long time waiting for neighbors to reach the same stage. This implies that the problem is not being load balanced well and a better algorithm is needed to reduce skew between machines. Nevertheless, Linde was able to achieve quite substantial speedup from his initial implementation of parallel SPH code.
